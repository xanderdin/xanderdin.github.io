My LibreCAD enhancements (October 2018 -- March 2019)

2019-03-25
Translations: RU

It is too fast running out of my head the memories of things I was investing my time and efforts into. So, after glancing on the history of my pull requests I decided to make a short report on them for the future remembrance. Most of the things described here are already merged into main LibreCAD branch, but some (at the moment of writing of this article) are still remaining as “Open” pull requests. Hope, they too would be merged soon. And here is the list of what I have done in the past few months to the benefits of LibreCAD.

✓ Added buttons to lock/unlock all layers at once in the Layer List (PR#1018).

✓ Added dxf2pdf command allowing running LibreCAD in console and converting a number of dxf files to corresponding one or many pdf files (PR#1023).

$ librecad dxf2pdf --help

Usage: librecad [options] <dxf_files>

dxf2pdf usage: librecad dxf2pdf [options] <dxf_files>

Print a bunch of DXF files to PDF file(s).

Examples:

  librecad dxf2pdf *.dxf    -- print all dxf files to pdf files with the same names.

  librecad dxf2pdf -o some.pdf *.dxf    -- print all dxf files to 'some.pdf' file.

Options:
  -h, --help                  Displays this help.
  -v, --version               Displays version information.
  -a, --fit                   Auto fit and center drawing to page.
  -c, --center                Auto center drawing on page.
  -k, --grayscale             Print grayscale.
  -m, --monochrome            Print monochrome (black/white).
  -p, --paper <WxH>           Paper size (Width x Height) in mm.
  -r, --resolution <integer>  Output resolution (DPI).
  -s, --scale <double>        Output scale. E.g.: 0.01 (for 1:100 scale).
  -o, --outfile <file>        Output PDF file.
  -t, --directory <path>      Target output directory.

Arguments:
  <dxf_files>                 Input DXF file(s)

✓ Created a patchset, which allows building of dxf2pdf as a standalone utility, that for its running does not require a running X server (X server libraries are still required to be present in the system though). This is useful if you need to run dxf2pdf on a remote server or, for example, in a Docker container.

✓ Fixed the meaning of the Row Spacing and Column Spacing fields (they were swapped) in the block properties dialog (PR#1028).

✓ Fixed an issue which led to block insertion always to layer 0, not to current active layer (PR#1030).

✓ Fixed an issue of ignoring the value of Scale Factor field during insertion of blocks (PR#1031).

✓ Added filter for filtering blocks in Block List, just like the filter in Layer List (PR#1044).

✓ Fixed a bug of losing parent properties by some of exploded entities (PR#1046).

✓ Fixed issues (#592, #605, #705) related to ignoring the value of “fixed” checkbox in print preview. Also made the paper always stay fully visible when switching to print preview (PR#1050).

✓ Added document modification status in window title and in open documents list (#1059).

✓ Added configuration parameters allowing to invert mouse wheel scrolling directions when scrolling with Shift or Ctrl keys pressed (PR#1061).

✓ Found and eliminated the roots of a couple of SEGFAULTs (PR#1032, PR#1052).

✓ Fixed an issue which led to disappearance of undone entities on further modifications (PR#1069).

✓ Added ability of selecting multiple layers or blocks in Layer List or Block List and applying a single action to all selected layers or blocks (PR#1071).

✓ Added buttons for refreshing content of Library Browser, eliminating therefore the need of restarting LibreCAD on adding new files to parts library (PR#1072).

✓ Added cache check before settings writing that reduces a lot of disk write operations and eliminates small freezings of the application on some functions (PR#1078).

✓ Fixed code (issue #1065), which on activated “Modify layer of selected entities” option led to layer change of not only selected block, but of all sub-entities of the block also. (PR#1081).

All those features and fixes allow me to get needed results with LibreCAD more quickly and productively. Hope, not only me but all LibreCAD users could benefit from those my efforts also!
