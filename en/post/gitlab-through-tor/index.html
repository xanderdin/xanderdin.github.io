GitLab through Tor

2019-01-03  
Translations: [ru]

When GitLab.com moved to Google Cloud Platform, which in turn is blocking connections from some places of our planet, I got troubles accessing my repositories and others hosted on GitLab.com. Having Tor Browser at hand, the quickies solution for me was using it for accessing GitLab.com. All I needed was running Tor Browser and telling git to use it as a proxy.

To make it work, first you need to tell git to use https for remote repository access (if not already configured so). I’m using origin_https name for that.

git remote add origin_https https://USER@gitlab.com/USER-OR-GROUP/REPO.git

And now it’s quite enough to assign the URL of local proxy (socks5://127.0.0.1:9150), accessable after running Tor Browser, to the HTTPS_PROXY environment variable, and run required git command.

HTTPS_PROXY="socks5://127.0.0.1:9150" git ... origin_https ...

Or in case of series of git commands:

export HTTPS_PROXY="socks5://127.0.0.1:9150"
git ... origin_https ...
git ... origin_https ...
...

You can also save the proxy URL to local or global git configuration:

git config (--global|--local) http.proxy socks5://127.0.0.1:9150

To unset the setting just run:

git config (--global|--local) --unset http.proxy

Another way is to set the value of remote.<name>.proxy. In my case:

git config remote.origin_https.proxy socks5://127.0.0.1:9150

And to unset:

git config --unset remote.origin_https.proxy

Good luck!
