Мои доработки LibreCAD (Октябрь 2018 -- Март 2019)

2019-03-25
Переводы: EN

Как-то быстро вылетают из моей головы те вещи, которым я посвящал своё время и над которыми трудился. Пролистал историю своих пул-реквестов и решил составить небольшой отчёт на память. Подавляющее большинство из перечисленного уже включено в основную ветку LibreCAD, однако (на момент написания данной статьи) некоторые всё ещё остаются в состоянии открытых пул-реквестов. Очень надеюсь, что и они будут вскоре приняты тоже. Итак, что было сделано мной за последние несполько месяцев для LibreCAD.

✓ Добавлены кнопки в Layer List, позволяющие блокировать или разблокировать одновременно все слои (PR#1018).

✓ Добавлена команда dxf2pdf для запуска LibreCAD в консоли и преобразования множества dxf файлов в соответствующие один или несколько pdf файлов (PR#1023).

$ librecad dxf2pdf --help

Usage: librecad [options] <dxf_files>

dxf2pdf usage: librecad dxf2pdf [options] <dxf_files>

Print a bunch of DXF files to PDF file(s).

Examples:

  librecad dxf2pdf *.dxf    -- print all dxf files to pdf files with the same names.

  librecad dxf2pdf -o some.pdf *.dxf    -- print all dxf files to 'some.pdf' file.

Options:
  -h, --help                  Displays this help.
  -v, --version               Displays version information.
  -a, --fit                   Auto fit and center drawing to page.
  -c, --center                Auto center drawing on page.
  -k, --grayscale             Print grayscale.
  -m, --monochrome            Print monochrome (black/white).
  -p, --paper <WxH>           Paper size (Width x Height) in mm.
  -r, --resolution <integer>  Output resolution (DPI).
  -s, --scale <double>        Output scale. E.g.: 0.01 (for 1:100 scale).
  -o, --outfile <file>        Output PDF file.
  -t, --directory <path>      Target output directory.

Arguments:
  <dxf_files>                 Input DXF file(s)

✓ Выпущен набор патчей, который позволяет собрать dxf2pdf отдельной утилитой, которая для своего запуска не требует запущенного работающего X сервера (библиотеки X сервера всё равно однако должны присутствовать в системе). Это удобно, если есть необходимость запускать dxf2pdf на удалённом сервере или, к примеру, в Docker контейнере.

✓ Исправлено назначение полей Row Spacing и Column Spacing (были перепутаны) в диалоге свойств блока (PR#1028).

✓ Исправлена ошибка, в результате которой блоки вставлялись не в текущий активный слой, а в слой “0” (PR#1030).

✓ Исправлена ошибка, в результате которой при вставке блока игнорировалось заданное значение поля Scale Factor (PR#1031).

✓ Добавлена возможность фильтрации блоков, по аналогии с фильтрацией слоёв (PR#1044).

✓ Исправлен баг, в результате которого часть “взорванных” сущностей теряло свойства родительского объекта (PR#1046).

✓ Исправлена проблема (#592, #605, #705) игнорирования флага масштаба “fixed” при предпечатном просмотре документа. Также сделано полное отображение листа при переключении в режим предпросмотра (PR#1050).

✓ Добавлено отображение статуса изменения документа в заголовке LibreCAD и списке редактируемых документов (#1059).

✓ Добавлены конфигурационные параметры, позволяющие инвертировать направление прокрутки мыши при использовании скролла совместно с клавишами Shift или Ctrl (PR#1061).

✓ Обнаружены и устранены причины нескольких SEGFAULT’ов (PR#1032, PR#1052).

✓ Устранена ошибка, в результате которой исчезали после модификации ранее отменённые сущности (PR#1069).

✓ Добавлена возможность множественного выбора слоёв и блоков в Layer List и Block List, позволяющего совершать действия одновременно с несколькими слоями или блоками (PR#1071).

✓ Добавлены кнопки обновления содержимого Library Browser, исключая таким образом необходимость перезапуска LibreCAD после добавления новых файлов в библиотеку компонентов (PR#1072).

✓ Добавлена проверка кэша перед записю конфигурации, которая значительно уменьшает количество дисковых операций записи и устраняет подтормаживание программы на некоторых операциях (PR#1078).

✓ Исправлен код (issue #1065), который в режиме “Modify layer of selected entities” приводил к изменению слоя не только для выделенного блока, но и для всех его подэлементов (PR#1081).

Все перечисленные доработки и исправления дают мне возможность использовать LibreCAD быстрее и продуктивнее . Надеюсь, проделанная работа принесёт пользу не только мне, но и всем пользователям LibreCAD!
